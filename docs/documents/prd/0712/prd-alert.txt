# 알림(Alert) 서비스 PRD

## 1. 목적 및 개요
- 리포트 상태 변경, 댓글, 시스템 이벤트 등 발생 시 사용자에게 실시간 알림 제공
- WebSocket/Kafka 기반 실시간 알림, 알림 내역 조회 등

## 2. 상세 요구사항
- 알림 생성/전송/조회/읽음처리 API 완전 구현
- WebSocket/Kafka 연동
- 알림 유형(리포트, 댓글, 시스템 등) 구분
- 사용자별 알림 구독/수신 관리

## 3. 동작 플로우
1. 이벤트 발생(리포트 상태 변경 등)
2. 알림 생성 및 전송(WebSocket/Kafka)
3. 사용자 앱/웹에서 알림 수신 및 표시
4. 알림 내역 조회/읽음 처리

## 4. 예외/에러 처리
- 알림 전송 실패, 연결 끊김, 권한 없음 등
