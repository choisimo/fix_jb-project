## 1. 개요

본 문서는 전북 현장 보고 앱의 사용자 경험(UX) 향상과 핵심 기능 개선을 위한 요구사항을 정의합니다. 주요 개선 사항은 보고서 목록 필터링 기능 추가, 로그인 상태 유지, 사용자 프로필 사진 기능 도입에 중점을 둡니다.

---

## 2. 목표

- **사용 편의성 증대**: 사용자가 원하는 정보에 더 빠르고 쉽게 접근할 수 있도록 UI/UX를 개선합니다.
- **개인화 기능 강화**: 사용자가 자신의 프로필을 꾸밀 수 있는 기능을 제공하여 앱에 대한 애착을 높입니다.
- **인증 과정 간소화**: 앱을 재시작할 때마다 로그인해야 하는 불편함을 해소하여 사용자 만족도를 높입니다.

---

## 3. 기능 요구사항

### 3.1. 보고서 목록 필터링 기능 추가

- **위치**: 메인 '보고서' 탭 > 보고서 목록 상단
- **기능 상세**:
    - 현재 '내 프로필 > 내가 작성한 보고서 목록'에 구현된 필터링 기능을 메인 보고서 목록에도 동일하게 적용합니다.
    - 사용자는 필터 아이콘을 통해 상태(예: 접수, 처리 중, 완료, 반려) 및 카테고리별로 전체 보고서 목록을 필터링할 수 있어야 합니다.
    - 필터 UI 및 동작 방식은 기존 기능과 일관성을 유지해야 합니다.

### 3.2. 로그인 상태 영구 유지

- **기능 상세**:
    - 사용자가 로그인을 한 번 완료하면, 앱을 완전히 종료(Terminate)했다가 다시 실행해도 로그인 상태가 유지되어야 합니다.
    - 인증 토큰(예: Access Token, Refresh Token)은 디바이스의 안전한 저장소(flutter_secure_storage 등)에 저장되어야 합니다.
    - 사용자가 명시적으로 '로그아웃'을 하기 전까지는 로그인 정보가 유지되어야 합니다.
    - 앱 업데이트 시에도 로그인 정보가 유지되는 것을 목표로 합니다.

### 3.3. 사용자 프로필 사진 기능 도입

- **위치**: '내 프로필' 화면
- **기능 상세**:
    - 프로필 화면에 기본 프로필 이미지 또는 사용자가 설정한 이미지를 표시하는 영역을 추가합니다.
    - 사용자는 해당 영역을 클릭하여 프로필 사진을 변경할 수 있어야 합니다.
    - 사진 변경 옵션은 다음 두 가지를 제공합니다.
        1.  카메라로 새로 촬영하기
        2.  갤러리에서 기존 이미지 선택하기
    - 업로드된 이미지는 서버에 저장되고, 다른 사용자(예: 관리자)도 해당 사용자의 프로필 사진을 볼 수 있어야 합니다.
    - 이미지 업로드 시 적절한 크기로 리사이징 및 압축하여 서버 부하와 데이터 사용량을 최소화해야 합니다.

## 4. 범위 외 사항 (Non-Goals)

- 본 요구사항에 명시되지 않은 모든 기능은 현재 상태를 유지합니다.
- UI/UX의 전반적인 리디자인은 포함되지 않습니다.
- 관리자 웹 대시보드의 기능 변경은 포함되지 않습니다.
- 필터링, 로그인, 프로필 사진 외 다른 기능의 로직 변경은 진행하지 않습니다.
